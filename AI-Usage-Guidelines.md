# AI Usage Guidelines

## Research notes

AI tools can be useful in a cybersecurity or governance role, but they need to be used carefully.

For this type of role, AI tools could be used for things like:

- summarising general concepts
- helping structure notes or documentation
- improving clarity in writing
- brainstorming questions to ask
- explaining security or compliance topics at a high level
- creating checklists or draft templates

The main benefit is that AI can save time and help organise information. It can also be useful when learning a new topic, because it can explain things in different ways.

The risks are also important. AI can give incorrect or outdated information, miss important context, or make something sound confident even when it is wrong. There is also a privacy risk if confidential information is pasted into an AI tool.

Information that should never be entered into AI tools includes:

- user data
- company confidential information
- credentials or API keys
- source code unless approved
- internal security findings
- private documents
- customer information
- anything from internal systems that is not approved for external use

To fact-check AI-generated content, I should compare it against trusted sources, official documentation, company policies, and my own understanding. I should not copy AI output without reviewing it properly.

## Reflection

I should use AI when it helps me learn, organise my thoughts, or improve the clarity of non-sensitive work. I should rely on my own skills when making decisions, handling company information, reviewing security issues, or submitting final work.

To avoid over-relying on AI, I should treat it as a support tool, not as the source of truth. If I use it, I still need to understand the answer, check whether it is accurate, and edit it so it matches the actual task.

For data privacy, I will avoid entering any Focus Bear internal information into AI tools unless I am clearly allowed to do so. If I am unsure, I will ask first. I will also avoid using AI for anything involving user data, credentials, code, internal documents, security findings, or confidential business information.

## Task I tried with AI

One task AI can help with is structuring notes or a reflection. For example, if I have rough points about a topic, AI can help turn them into clearer sections or suggest a better layout.

However, the output still needs editing and fact-checking. It may sound polished but still be too generic, too confident, or not fully accurate. I would need to review it, remove anything that does not match my understanding, and make sure it does not include sensitive information.

## Best practice I will follow

The main best practice I will follow is:

**Do not enter confidential, private, or internal Focus Bear information into AI tools unless I have clear approval.**

I will also make sure that any AI-assisted work is reviewed, edited, and understood by me before I use it.
